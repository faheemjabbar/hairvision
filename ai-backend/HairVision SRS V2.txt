1. INTRODUCTION

1.1 Purpose

The system requirement specification describes the functional and non-functional requirements
for HairVision. The purpose of this document is to specify detailed requirements for the
AI-Powered Hair & Scalp Analyzer. This is the first version of this SRS. The intended readers of
this document are system analysts, designers, project developers, and the project panel. The
developer can use this document for traceability of functions implemented.


1.2 Scope

The aim of this project is to design and develop an AI-based web application that allows users
to upload a photo of their scalp or hair for automated analysis. This system will benefit
individuals seeking early detection of hair issues and personalized care. The system processes
the image to detect conditions such as baldness, hair quality, and scalp health. It displays a
condition summary, confidence level, and relevant care suggestions. The scope is limited to
image analysis and recommendation; it does not replace medical treatment by a doctor.

2. THE OVERALL DESCRIPTION

2.1 Product Perspective
This application is a web-based system that works independently. It does not require complex
external hardware other than a device with a camera or storage to upload images. The system
relies on an AI model integrated into the backend to perform the analysis. The most important
parts of the system are the User (who uploads images) and the Admin (who manages the
system).


2.2 Product Functions
HairVision is a major utility for personal grooming and health monitoring. It provides the facility
to the user to register and login. After logging in, the user can upload an image of their hair or
scalp. The system uses AI to analyze the image and returns a result containing the condition
(e.g., thinning, oily scalp), a confidence percentage, and care suggestions. It maintains a history
of user analyses.


2.3 User Characteristics
There are two categories of users:

   1.​ User/Visitor: Individuals interested in analyzing their hair health. They do not require
       high technical skills, only the ability to browse the web and upload photos.
   2.​ Administrator: A person responsible for the management and maintenance of the
       portal. The administrator requires technical knowledge to manage the database and user
       records.


2.4 General Constraints
This is a web-based application. The interface is user-friendly and requires no special training.
The interface uses the English language. Users must register to save their analysis history. The
accuracy of the result depends on the quality of the uploaded image.


2.5 Assumptions and Dependencies
The primary dependency of HairVision is an internet connection. The AI model assumes that the
uploaded images are clear and well-lit. All user data is saved in a secure environment.

3. SPECIFIC REQUIREMENTS

3.1 External Interface Requirements

3.1.1 System Interfaces
HairVision interfaces with a database (e.g., MySQL or PostgreSQL) to store user profiles and
reports. It also interfaces with the AI model server (backend) to process images.


3.1.2 Interfaces
The web interface is designed to be user-friendly using GUI principles. It provides a clean
dashboard for users to upload images and view reports.


3.1.3 Hardware Interfaces
The system requires a computer, laptop, or smartphone with internet access. A camera or
storage device is required to provide the image for analysis.


3.1.4 Software Interfaces
   ●​   Operating System: Windows, Linux, or macOS.
   ●​   Text Editor: VS Code, Sublime Text, or Notepad++.
   ●​   Server: Apache or Nginx for hosting the web app.
   ●​   Database: MySQL or SQLite.


3.1.5 Communications Interfaces
HTTP/HTTPS protocols will be used to communicate between the client (browser) and the web
server.


3.2 Functional Requirements

3.2.1 Registration
   ●​   Introduction: Users must register to access the full features of the system.
   ●​   Inputs: Name, Email, Password.
   ●​   Processing: System validates inputs and creates a new user record in the database.
   ●​   Output: User is successfully registered and redirected to login.

3.2.2 Login
  ●​   Introduction: Users login to access their dashboard.
  ●​   Inputs: Email, Password.
  ●​   Processing: System verifies credentials against the database.
  ●​   Output: Successful login or error message.


3.2.3 Upload Image
  ●​   Introduction: The core feature where users provide data for analysis.
  ●​   Inputs: Image file (JPG, PNG).
  ●​   Processing: System accepts the file and sends it to the AI model.
  ●​   Output: Image uploaded successfully.


3.2.4 Analyze Condition
  ●​   Introduction: Automated analysis of the scalp/hair.
  ●​   Inputs: Uploaded Image.
  ●​   Processing: AI model processes the image to detect features like baldness or dandruff.
  ●​   Output: Analysis result generated.


3.2.5 View Report
  ●​   Introduction: Displaying results to the user.
  ●​   Inputs: Request to view result.
  ●​   Processing: System fetches analysis data and recommended tips.
  ●​   Output: Display of condition summary, confidence level, and care suggestions.


3.2.6 Feedback
  ●​   Introduction: Users can provide feedback on the analysis accuracy.
  ●​   Inputs: Text comments, Rating.
  ●​   Processing: Feedback stored in the database for Admin review.
  ●​   Output: Thank you message.


3.2.7 Manage Users (Admin)
  ●​   Introduction: Admin can view or remove users.
  ●​   Inputs: Admin credentials.
  ●​   Processing: Delete or update user records.
  ●​   Output: User list updated.

3.5 Non-Functional Requirements
   ●​   3.5.1 Performance: The system should generate analysis results within 5-10 seconds.
   ●​   3.5.2 Reliability: The AI model should provide consistent results for similar images.
   ●​   3.5.3 Availability: The system should be available 24/7.
   ●​   3.5.4 Security: User passwords must be encrypted; images should be private.
   ●​   3.5.5 Maintainability: The code structure should be modular for easy updates.

5. APPENDICES & CONCLUSION
5.1 Conclusion

The HairVision System has been designed to address the growing need for accessible and
early detection of scalp and hair conditions. By leveraging advanced AI image processing and a
user-friendly web interface, the system successfully bridges the gap between patients and
immediate trichological advice.

This SRS document has defined the complete scope, functional requirements, and behavioral
models of the system. The analysis models (Sequence, DFD, and State-Transition Diagrams)
confirm that the system’s logic is sound and ready for the implementation phase. The proposed
database structure ensures data integrity for user records, analysis reports, and feedback.

Ultimately, HairVision aims to empower users to take proactive control of their hair health
through technology that is fast, private, and reliable.
